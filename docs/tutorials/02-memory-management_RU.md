# Руководство 2: Управление памятью

Argentum имеет сложную многослойную систему памяти. Это руководство подробно рассматривает каждый слой, показывает, как работать с ними напрямую, и объясняет автоматизацию, которая поддерживает память полезной со временем.

**Примерное время:** 20 минут  
**Требования:** Работающий экземпляр Argentum (из [Руководства 1](./01-first-agent_RU.md))

---

## Четыре слоя памяти

Argentum организует память на четыре слоя, каждый со своей ролью:

| Слой | Технология | Назначение | Время жизни |
|---|---|---|---|
| **Сессия** | В оперативной памяти | Контекст текущего разговора | Пока сессия активна |
| **Семантическая** | SQLite + FTS5 | Факты, индексированные по смыслу | Постоянно до очистки |
| **Граф знаний** | SQLite + граф | Связи между сущностями | Постоянно до очистки |
| **Markdown** | Файлы на диске | Читаемая справочная информация | Постоянно, управляется файлами |

Сообщения проходят через все слои при обработке. Понимание каждого из них помогает эффективно использовать и отлаживать систему.

---

## Слой 1: Сессионная память

Сессионная память хранит текущий разговор. Она поддерживается в оперативной памяти во время сессии и сохраняется в `data/sessions.db` при завершении сессии или через интервалы контрольных точек.

### Как это работает

Когда вы отправляете сообщение с известным `userId`, агент извлекает недавнюю историю разговора этого пользователя и включает её в контекст LLM. Это то, что делает многоходовые разговоры возможными.

```bash
# Начать разговор
curl -X POST http://localhost:18789/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Я изучаю TypeScript", "userId": "alice"}'
# sessionId: sess_abc123

# Продолжить разговор — сессия автоматически загружается
curl -X POST http://localhost:18789/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Что мне изучать дальше?", "userId": "alice"}'
# Агент знает, что вы изучаете TypeScript
```

### Настройка сессий

```json
{
  "features": {
    "sessions": {
      "enabled": true,
      "maxHistory": 50,
      "checkpointInterval": 5
    }
  }
}
```

### Управление сессиями

```bash
# Список всех сессий
argentum sessions list

# Просмотр конкретной сессии
argentum sessions view sess_abc123

# Очистка сессии (сброс разговора)
argentum sessions clear sess_abc123

# Экспорт сессии в JSON
argentum sessions export sess_abc123 > conversation.json
```

## Слой 2: Семантическая память

Семантическая память хранит факты и информацию в SQLite с полнотекстовым поиском. Записи индексируются так, чтобы агент мог найти релевантный контекст даже когда поисковые запросы не точно совпадают с сохранённым текстом.

### Явное сохранение воспоминаний

```bash
curl -X POST http://localhost:18789/memory/store \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Алиса предпочитает тёмную тему в IDE",
    "tags": ["предпочтение", "разработка"],
    "userId": "alice"
  }'
```

Ответ:

```json
{
  "success": true,
  "data": {
    "id": "mem_abc123",
    "createdAt": "2026-03-23T10:00:00Z"
  }
}
```

### Поиск в семантической памяти

```bash
# Базовый поиск
curl "http://localhost:18789/memory/search?q=тёмная%20тема"

# С лимитом и порогом
curl "http://localhost:18789/memory/search?q=предпочтения&limit=5&threshold=0.7"
```

Ответ:

```json
{
  "success": true,
  "data": {
    "results": [
      {
        "id": "mem_abc123",
        "content": "Алиса предпочитает тёмную тему в IDE",
        "score": 0.94,
        "createdAt": "2026-03-23T10:00:00Z",
        "tags": ["предпочтение", "разработка"]
      }
    ],
    "total": 1
  }
}
```

### Как агент её использует

При обработке сообщения агент запрашивает семантическую память для релевантного контекста. Модуль `self-evolving-memory` также автоматически захватывает факты из разговоров. Например, если вы скажете:

> "Я начал стажировку в Binarwelt"

Argentum автоматически сохраняет это как запись с тегами `["факт", "работа", "стажировка"]`.

### Теги памяти

Теги помогают организовывать и фильтровать воспоминания. Используйте их при явном сохранении:

```bash
curl -X POST http://localhost:18789/memory/store \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Дедлайн проекта — 30 марта",
    "tags": ["дедлайн", "проект"],
    "userId": "alice"
  }'
```

Запрос по тегу:

```bash
curl "http://localhost:18789/memory/search?q=дедлайн&tags=проект"
```

## Слой 3: Граф знаний

Граф знаний хранит сущности и их отношения как направленный граф. Это позволяет агенту делать выводы на основе связанных фактов.

### Сущности и отношения

Сущность — это узел с типом и свойствами:

```json
{
  "id": "entity_001",
  "type": "person",
  "name": "Алиса",
  "properties": {
    "location": "Сан-Франциско, США",
    "timezone": "America/Los_Angeles"
  }
}
```

Отношение — это направленное ребро между сущностями:

```json
{
  "from": "entity_001",
  "type": "работает_в",
  "to": "entity_002"
}
```

### Как граф строится

Агент автоматически извлекает сущности и отношения из разговоров. Когда вы говорите:

> "Я работаю в Acme Corp"

Argentum создаёт:
- Сущность: `Acme Corp` (тип: компания)
- Сущность: `Алиса` (тип: человек)
- Отношение: Алиса → `работает_в` → Acme Corp

### Запросы к графу

```bash
# Получить всю информацию о конкретной сущности
curl "http://localhost:18789/memory/graph?entity=Алиса"
```

```json
{
  "success": true,
  "data": {
    "entities": [
      {
        "id": "entity_001",
        "type": "person",
        "name": "Алиса",
        "properties": {
          "location": "Сан-Франциско, США",
          "timezone": "America/Los_Angeles"
        }
      }
    ],
    "relations": [
      { "from": "entity_001", "type": "работает_в", "to": "entity_002" },
      { "from": "entity_001", "type": "говорит_на", "to": "entity_003" },
      { "from": "entity_001", "type": "интересуется", "to": "entity_004" }
    ]
  }
}
```

### Ручные операции с графом

Добавить сущность напрямую:

```bash
curl -X POST http://localhost:18789/memory/graph/entity \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Argentum",
    "type": "проект",
    "properties": {
      "version": "0.0.2",
      "language": "TypeScript"
    }
  }'
```

Добавить отношение:

```bash
curl -X POST http://localhost:18789/memory/graph/relation \
  -H "Content-Type: application/json" \
  -d '{
    "from": "entity_001",
    "type": "создал",
    "to": "entity_005"
  }'
```

### Обход графа

Граф знаний поддерживает запросы обхода. Найти всех людей, которые работают в том же месте, что и Алиса:

```bash
curl "http://localhost:18789/memory/graph/traverse?from=entity_001&type=работает_в&depth=1"
```

## Слой 4: Markdown память

Markdown-файлы на диске служат вручную курируемым, читаемым слоем памяти. Argentum наблюдает за каталогом `data/memory/` и автоматически загружает любые `.md` файлы в контекст.

### Создание файлов памяти

```bash
mkdir -p data/memory

cat > data/memory/personal-context.md << 'EOF'
# Личный контекст

## Об Алисе
- Full-stack разработчик, изучает TypeScript
- Живёт в Сан-Франциско, США
- Доступен для удалённой работы
- часовой пояс: America/Los_Angeles

## Текущие проекты
- Создаёт AI-агент фреймворк (Argentum)
- Поиск работы программистом

## Навыки
- TypeScript, JavaScript, Java, Python
- Linux, Docker, Git
- Русский (родной), Английский (B2-C1), Эстонский (B1-B2)

## Цели на 2026
- Найти работу программистом
- Изучить Rust
- Участвовать в open source
EOF
```

Изменения в этом файле подхватываются сразу — перезапуск не нужен.

### Как Markdown память интегрируется

Модуль `markdown-memory` парсит эти файлы и внедряет релевантные разделы в контекст агента. Это особенно полезно для:
- Фактов, которые никогда не должны быть забыты или сжаты
- Вручную курируемого контекста из других систем
- Долгосрочных целей и предпочтений

### Именование файлов памяти

Файлы обрабатываются в алфавитном порядке. Используйте префиксы для управления порядком загрузки:

```
data/memory/
├── 00-identity.md      # Загружается первым — базовая идентичность
├── 01-preferences.md   # Пользовательские предпочтения
├── 02-projects.md      # Текущие проекты
└── 03-goals.md         # Цели и устремления
```

## Self-Evolving Memory

Модуль `self-evolving-memory` автоматизирует обслуживание памяти. Он работает в фоне и выполняет три задачи:

### 1. Консолидация

Когда семантическая память становится большой, похожие записи объединяются:

```
До:  100 записей о "встречах по проекту"
После: 3 консолидированные записи о встречах по проекту
```

Это поддерживает использование контекстного окна управляемым без потери информации.

### 2. Обнаружение паттернов

Модуль сканирует повторяющиеся темы и создаёт сводные записи:

```
Обнаружен паттерн: "еженедельный синк по проекту каждую понедельник"
Создана сводка: "У пользователя повторяющиеся встречи по понедельникам"
```

### 3. Затухание релевантности

Малозначимые записи, к которым долго не обращались, постепенно теряют приоритет:

```json
{
  "features": {
    "self-evolving-memory": {
      "enabled": true,
      "consolidationThreshold": 500,
      "decayAfterDays": 30,
      "minRelevanceScore": 0.2
    }
  }
}
```

## Статистика и обслуживание памяти

### Просмотр статистики

```bash
argentum memory stats
```

Вывод:

```
Semantic Memory:  1247 записей
Knowledge Graph:   342 сущности, 891 отношение
Markdown Files:      4 файла загружено
Session Memory:      8 активных сессий
Коэффициент сжатия:   73%
```

### Очистка старых записей

```bash
# Удалить воспоминания старше даты
argentum memory purge --before 2026-01-01

# Очистить по тегу
argentum memory purge --tag temporary

# Показать, что будет удалено, без реального удаления
argentum memory purge --before 2026-01-01 --dry-run
```

### Экспорт и импорт

Экспортировать все воспоминания для резервной копии или миграции:

```bash
argentum memory export --format json > memories-backup-$(date +%Y%m%d).json
```

Импорт:

```bash
argentum memory import memories-backup-20260318.json
```

## Отладка памяти

### Проверить, что агент извлекает

Включите debug-логирование, чтобы видеть запросы к памяти:

```bash
AGCLAW_LOG_LEVEL=debug argentum gateway start
```

Ищите строки:

```
[memory] Searching semantic: query="TypeScript" limit=5 threshold=0.6
[memory] Retrieved 3 results (score >= 0.6)
[memory] Graph query: entity=Alice depth=2
[memory] Graph results: 4 entities, 7 relations
```

### Проверить, что воспоминание сохранилось

```bash
curl "http://localhost:18789/memory/search?q=Алиса%20TypeScript"
```

### Проверить целостность графа знаний

```bash
# Список сущностей определённого типа
curl "http://localhost:18789/memory/graph?type=person"

# Найти осиротевшие сущности (без связей)
curl "http://localhost:18789/memory/graph?orphans=true"
```
