# Урок 1: Ваш первый агент

*Примерное время: 15 минут*

В этом уроке вы создадите своего первого агента AG-Claw с нуля, настроите его и проведёте с ним первую беседу.

---

## Что вы узнаете

- Как инициализировать экземпляр AG-Claw
- Как настроить личность и возможности агента
- Как подключить канал связи (Telegram)
- Как убедиться, что всё работает

---

## Шаг 1 — Инициализация

Если вы ещё не сделали это, клонируйте и установите AG-Claw:

```bash
git clone https://github.com/AG064/ag-claw.git
cd ag-claw
npm install
npm link
```

Инициализируйте в новой директории:

```bash
mkdir my-agent && cd my-agent
agclaw init
```

Это создаст:
- `agclaw.json` — ваш файл конфигурации
- `data/` — хранилище базы данных
- `.agclaw/` — состояние runtime

---

## Шаг 2 — Установите API-ключ

AG-Claw нужен LLM для работы. Получите API-ключ от [OpenRouter](https://openrouter.ai/) (рекомендуется — поддерживает множество моделей).

```bash
export OPENROUTER_API_KEY=sk-or-v1-...
```

Добавьте его постоянно, записав строку в профиль оболочки (`~/.zshrc` или `~/.bashrc`):

```bash
echo 'export OPENROUTER_API_KEY=sk-or-v1-...' >> ~/.zshrc
source ~/.zshrc
```

---

## Шаг 3 — Настройте агента

Откройте `agclaw.json` и настройте его:

```json
{
  "name": "Мой первый агент",
  "server": {
    "port": 3000
  },
  "model": {
    "provider": "openrouter",
    "defaultModel": "anthropic/claude-sonnet-4-20250514"
  },
  "features": {
    "sqlite-memory": { "enabled": true },
    "semantic-search": { "enabled": true },
    "cron-scheduler": { "enabled": true },
    "morning-briefing": { "enabled": false },
    "evening-recap": { "enabled": false },
    "webchat": { "enabled": true }
  },
  "channels": {
    "telegram": { "enabled": false },
    "webchat": { "enabled": true, "port": 3001 }
  }
}
```

Ключевые choices:
- **Name**: Дайте агенту имя
- **Model**: `claude-sonnet-4` быстрая и способная; попробуйте `claude-opus-4` для сложных задач
- **Memory**: Держите `sqlite-memory` и `semantic-search` включёнными — это мозг
- **Channels**: Начните с `webchat` для лёгкого тестирования; добавьте Telegram позже

---

## Шаг 4 — Запуск и проверка

Запустите шлюз:

```bash
agclaw gateway start --port 3000
```

Проверьте здоровье:

```bash
curl http://localhost:3000/health
```

Вы должны увидеть:

```json
{
  "status": "ok",
  "version": "0.2.0",
  "features": {
    "total": 59,
    "active": 6,
    "unhealthy": []
  }
}
```

---

## Шаг 5 — Тест через Webchat

Откройте `http://localhost:3001` в браузере (или отправьте тестовое сообщение):

```bash
curl -X POST http://localhost:3000/chat \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Привет! Как тебя зовут?",
    "userId": "test-user"
  }'
```

Агент должен ответить своим именем. Теперь попробуйте:

```
Пользователь: Запомни, что мой любимый язык программирования — Rust.
Агент: Записал! Ты запомнил, что твой любимый язык программирования — Rust.

Пользователь: Какой мой любимый язык программирования?
Агент: Твой любимый язык программирования — Rust.
```

Система памяти работает — агент запомнил то, что вы ему сказали.

---

## Шаг 6 — Подключение Telegram (опционально)

Хотите общаться через Telegram? Вот как это сделать:

1. Откройте Telegram и начните чат с [@BotFather](https://t.me/BotFather)
2. Отправьте `/newbot` и следуйте инструкциям
3. Скопируйте токен, который даст вам BotFather
4. Обновите `agclaw.json`:

```json
{
  "channels": {
    "telegram": {
      "enabled": true,
      "token": "7123456789:AAFxxxxxxxxxxxxxxxxxxxxxx"
    }
  }
}
```

5. Перезапустите: `agclaw gateway restart`
6. Откройте Telegram и найдите своего бота — отправьте `/start`

## Решение проблем

| Проблема | Решение |
|---|---|
| Ошибка "API key not set" | Убедитесь, что `OPENROUTER_API_KEY` экспортирован перед запуском |
| Пустой ответ от агента | Проверьте `agclaw gateway logs` на наличие ошибок |
| Webchat не загружается | Убедитесь, что `webchat` включён в features и channels |
| Telegram не отвечает | Проверьте токен; посмотрите `agclaw gateway logs` |
| Порт уже используется | Измените порт: `agclaw gateway start --port 4000` |
