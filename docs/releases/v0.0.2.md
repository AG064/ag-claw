# Argentum v0.0.2

This release focuses on getting Argentum into an installable Windows shape while keeping the security fixes and onboarding flow from the development branch.

## Highlights

- Added Windows release artifacts:
  - `argentum-v0.0.2-win-x64.exe` graphical setup executable
  - `argentum-v0.0.2-win-x64.msi` Windows installer
  - `argentum-v0.0.2-win-x64-portable.exe` portable CLI executable
- Added release automation for Windows, Linux, and macOS binaries.
- Added checksum files for verifying downloaded artifacts.
- Added a standard Windows setup wizard with license agreement, install location selection, Program Files install, and Start Menu shortcut.
- Hardened API gateway authentication with rate limiting for protected routes.
- Improved onboarding so first-time setup writes a secure, loopback-only default profile.
- Kept secrets out of generated YAML config by writing sensitive values to `.env`.

## Windows Setup

Download `argentum-v0.0.2-win-x64.exe` for the normal graphical setup flow, or use the `.msi` if you prefer Windows Installer directly. Use `argentum-v0.0.2-win-x64-portable.exe` only when you specifically want the console CLI without installing.

When launched without arguments on Windows, the portable executable now opens the first-run setup flow instead of printing help and immediately closing the terminal window.

## Verification

- TypeScript build passes.
- Unit and onboarding tests pass.
- Windows release workflow smoke tests cover `--version`, non-interactive onboarding, no-argument packaged launch behavior, MSI creation, and setup `.exe` creation.
- GitHub Actions release workflow publishes the setup `.exe`, portable `.exe`, `.msi`, and checksum files.
