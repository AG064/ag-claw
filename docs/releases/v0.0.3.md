# Argentum v0.0.3

Argentum v0.0.3 is the first release shaped around the new Argentum identity, a safer local-first runtime model, and real installable desktop artifacts. It brings the project closer to a normal app experience while keeping the CLI and automation workflows available for developers.

## Install

Recommended downloads:

- `argentum-v0.0.3-win-x64.exe` - Windows graphical setup flow.
- `argentum-v0.0.3-win-x64.msi` - Windows Installer package.
- `argentum-v0.0.3-win-x64-portable.exe` - portable Windows CLI build.
- Desktop bundles for Windows, Linux, macOS Apple Silicon, and macOS Intel from the Desktop Builds release assets.
- Portable CLI binaries for Linux x64 and macOS x64.

Checksum files are included with release artifacts so downloads can be verified before installation.

## Highlights

- Rebranded AG Claw to Argentum across the product surface, package metadata, documentation, installer naming, executable metadata, and desktop shell.
- Bumped and synchronized the repository to `v0.0.3` with CI checks that catch future version drift.
- Added a Tauri desktop app scaffold with Argentum branding, product icon support, onboarding-first flow, and a setup save bridge into the native app.
- Added cross-platform desktop build automation for Linux, Windows, macOS Apple Silicon, and macOS Intel.
- Added Windows installer automation for setup `.exe`, `.msi`, and portable `.exe` artifacts.
- Added a professional release workflow that publishes release notes, portable binaries, installers, desktop bundles, and checksum files from a tag.

## Security

- Added a workspace-boundary policy so file access stays inside the user-selected workspace unless explicitly allowed.
- Added a capability broker for privileged file and shell actions, with audit entries and secret redaction.
- Kept dangerous tools disabled by default and routed privileged built-in tools through the capability broker.
- Hardened API gateway authorization with rate limiting on protected routes.
- Kept public chat channels disabled by default and ensured first-run onboarding writes a loopback-only profile unless the user opts in.
- Ensured generated YAML config does not contain secrets; sensitive values are written outside YAML.
- Added and verified security automation for dependency audit, secret scanning, Trivy container scanning, CodeQL compatibility, and OpenSSF Scorecard SARIF upload.

## Onboarding And Desktop

- Added an onboarding-first desktop UI with workspace location, runtime, provider, channel, and security profile choices.
- Added native setup persistence for desktop onboarding, including config, data, logs, audit directory, and secrets placeholder files.
- Fixed packaged Windows launch behavior so the app does not immediately close after first-run setup.
- Added proper macOS `.icns`, Windows `.ico`, and shared PNG icon assets for native desktop bundling.
- Kept the CLI available for terminal-first users and non-Windows environments.

## Breaking Changes

- The public product name is now Argentum.
- Legacy `ag-claw` / `agclaw` command and branding references were removed where they were product-owned.
- Configuration now prefers `argentum.json`; legacy config names are intentionally not treated as the primary path.

## Notes

The desktop UI in v0.0.3 is the foundation for the full Argentum application experience: onboarding, setup persistence, shell layout, and native packaging are in place. The CLI remains the most complete execution surface while the desktop app grows into the full agent interface.
