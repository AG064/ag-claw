# Argentum v0.0.4

Argentum v0.0.4 fixes the Windows release packaging problem where the prominently named installer assets launched the terminal CLI instead of the desktop interface. This release makes the GUI installer the default user path and keeps CLI binaries explicitly optional.

## Install

Recommended downloads:

- `Argentum_0.0.4_x64-setup.exe` - Windows desktop setup executable.
- `Argentum_0.0.4_x64_en-US.msi` - direct Windows Installer package.
- `Argentum_0.0.4_amd64.AppImage`, `.deb`, or `.rpm` - Linux desktop packages.
- `Argentum_0.0.4_x64.dmg` or `Argentum_0.0.4_aarch64.dmg` - macOS desktop packages.
- `argentum-cli-v0.0.4-win-x64.exe` - optional Windows terminal CLI.
- `argentum-v0.0.4-linux-x64` and `argentum-v0.0.4-macos-x64` - optional portable CLI binaries.

## Fixed

- Windows setup and MSI assets are now produced by the Tauri desktop workflow, so launching the installed app opens the Argentum GUI instead of a terminal diagnostics screen.
- Windows CLI assets are renamed with `argentum-cli-` so they are visibly terminal-only.
- `npm run package:win` now builds the GUI desktop app. Use `npm run package:win:cli` for the optional portable CLI.
- Packaged Windows CLI binaries no longer hijack no-argument execution into the old terminal launch flow; they show normal CLI help unless a command is provided.
- Release workflows no longer publish the CLI `.exe`/`.msi` as the graphical Windows installer.
- Version synchronization now covers Tauri and Cargo desktop metadata.

## Notes

If you installed v0.0.3 and saw only a terminal window, install v0.0.4 using `Argentum_0.0.4_x64-setup.exe`. The old `argentum-v...-win-x64.exe` naming was CLI packaging and should not be used as the desktop installer.
