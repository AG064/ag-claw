# Argentum v0.0.5

Argentum v0.0.5 is the first release pass focused on making the desktop MVP feel like a coherent product: stable ChatGPT/OpenAI and MiniMax provider paths, MiniMax usage visibility, cleaner gateway output, reasoning-block display controls, tighter chat history behavior, and explicit OS/hardware support notes.

## Install

Recommended downloads:

- `Argentum_0.0.5_x64-setup.exe` - Windows desktop setup executable.
- `Argentum_0.0.5_x64_en-US.msi` - direct Windows Installer package.
- `Argentum_0.0.5_amd64.AppImage`, `.deb`, or `.rpm` - Linux desktop packages.
- `Argentum_0.0.5_x64.dmg` or `Argentum_0.0.5_aarch64.dmg` - macOS desktop packages.
- `argentum-cli-v0.0.5-win-x64.exe` - optional Windows terminal CLI.
- `argentum-v0.0.5-linux-x64` and `argentum-v0.0.5-macos-x64` - optional portable CLI binaries.

## Stable Providers

Stable providers in v0.0.5:

- ChatGPT / OpenAI is stable for live desktop chat through API key auth and ChatGPT/Codex browser-account authorization.
- MiniMax is stable for live desktop chat through API key auth, with Token Plan usage checks and M2.7 guidance wired into Diagnostics and model context.

## Testing Providers

Testing providers in v0.0.5:

Anthropic Claude, Google Gemini, OpenRouter, NVIDIA, Groq, Ollama/local, and Custom endpoint remain in testing state. They are still selectable and testable, but they are not promoted as stable release routes yet.

## New

- MiniMax Token Plan usage now checks `https://www.minimax.io/v1/token_plan/remains`, shows remaining plan data in Diagnostics, includes reset cadence, and exposes the snapshot to the agent runtime context.
- MiniMax M2.7 best-practice guidance is included in runtime context: clear instructions, intent, examples, phased long work, state tracking, and compact context management.
- Chat responses now render Markdown and split provider-visible `<think>...</think>` or `<reasoning>...</reasoning>` blocks into a muted, collapsible reasoning panel above the final answer.
- Settings now include separate toggles for showing reasoning blocks in desktop Chat and sending reasoning blocks to Telegram.
- Chat history now stays history-based: newer conversations stay above older ones, and merely selecting a chat no longer moves it to the top.
- Conversation header, recent chat list, transcript, and composer are laid out so only the chat history scrolls during normal conversation.
- Chat delete controls are less visually loud and still require confirmation.
- Provider settings make ChatGPT/OpenAI and MiniMax stable while grouping all other provider presets under Testing.
- Gateway action output is more compact. Start/stop actions now keep terminal output brief, while Gateway Status carries PID, health URL, state, and log path details.
- The agent system context now explicitly instructs models to redact raw system prompt, hidden runtime instructions, API keys, tokens, and private profile fields.
- README and release notes now list supported OS targets, release assets, and hardware requirements.

## Supported OS

| Platform | Supported release target | Release assets | Runtime notes |
| --- | --- | --- | --- |
| Windows | Windows 10/11 x64 | `.exe`, `.msi`, optional CLI `.exe` | Uses Microsoft Edge WebView2. Windows 11 includes it; Windows 10 1803+ usually has it, and the installer can install it if missing. |
| Linux | Ubuntu 22.04+ x64 and compatible desktop distributions with WebKitGTK 4.1 | `.AppImage`, `.deb`, `.rpm`, optional portable CLI | Intended for Ubuntu/Debian/Fedora/Arch/openSUSE style desktop environments with the normal Tauri/WebKitGTK stack. |
| macOS | macOS 10.15+ on Intel and Apple Silicon | x64 and aarch64 `.dmg`, optional portable CLI | Uses system WKWebView. Apple Silicon users should prefer the `aarch64` DMG. |

## Hardware Requirements

Hardware requirements for v0.0.5:

| Level | Requirement |
| --- | --- |
| Minimum | 64-bit CPU, 4 GB RAM, 500 MB free disk space, working system webview, internet access for hosted providers. |
| Recommended | 4-core CPU, 8 GB RAM, 1 GB free disk space, stable broadband, and an SSD-backed workspace. |
| Gateway | Localhost port `3000` available by default. Provider/network access still depends on workspace configuration and approval. |
| Providers | ChatGPT/OpenAI usage follows the selected account or API plan. MiniMax Token Plan usage and reset cadence are visible in Diagnostics and model context. |

## Notes

- The Windows GUI installer is still the recommended first install path.
- CLI binaries remain optional and terminal-first.
- Provider usage limits depend on provider accounts and API plans. Argentum reports what the provider exposes through headers or usage endpoints.
